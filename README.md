# Goodnotes System Design & Architecture

## Project Overview
This project focuses on the **Software Engineering** phase of developing a digital note-taking application inspired by "Goodnotes". It covers the entire design lifecycle, from stakeholder identification to architectural patterns and UML modeling, ensuring a scalable and user-centric system.

##  Key Features Analyzed
- **User Management:** Secure authentication and profile management.
- **Note Organization:** Creating, editing, and tagging notes within a hierarchical structure.
- **Multimedia Integration:** Support for adding text, images, and audio to notes.
- **Subscription Model:** Managing payments and premium feature access.

##  System Architecture
- **Pattern:** Layered Architecture (chosen for its scalability and separation of concerns).
- **Design Diagrams:**
  - **Use Case Diagram:** Defines the interactions between Users, Admins, and the system.
  - **Sequence Diagram:** Illustrates the dynamic flow of operations like creating notes.
  - **Class Diagram:** Represents the static structure and relationships between system entities.
  - **Activity Diagram:** Shows the workflow of the application's main processes.

##  Tools Used
- **Modeling:** StarUML / Lucidchart (or mention the tool you used).
- **Documentation:** Microsoft Word / PDF.

## Contents
- `/diagrams`: Contains all UML diagrams (Use Case, Class, Sequence, etc.).
- `Project_Report.pdf`: The full theoretical study and software requirement specifications (SRS).

## Learning Outcomes
Through this project, I applied core Software Engineering principles, including:
- Requirement engineering (Functional & Non-functional).
- Architectural design decisions.
- Professional UML modeling to bridge the gap between requirements and implementation.
